Hexo+github actions完成个人博客的自动部署

This is the first blog! it is created by github-actions!
本人一直对个人博客比较感兴趣，之前也用过hexo进行过博客的编写，但由于其编写需要部署hexo环境，有点麻烦，而我又想在多端编写，所以后来也就放弃了。但最近发现github actions有这个自动部署的功能，在借鉴了网上的资料之后也终于搭建一个可以自动部署的hexo。

站点演示传送门

效果预览

Hexo

More info: Generating

github actions

github actions分为两个部分的配置文件，一个部分是新建博客，另一个是部署博客

实现原理

我首先对整个实现的原理做一个简单的介绍，也让大家能明白每一步是在干什么。要实现这样的效果首先需要创建两个github仓库，一个仓库用于存放hexo生成的博客源码目录（下面统称为源码仓库），建议为私有的，保护隐私；另一个仓库用于存放hexo生成的网页渲染文件（下面统称为网页仓库）。

通过github actions监控源码仓库的变动，自动在容器中更新文件，再更新到网页仓库，以达到自动部署的效果。

注意：网页仓库是利用github page的功能，其仓库名必须是用户名.github.io, 例如我的是yuyanghou1230.github.io.

新建博客功能

新建博客是借助github actions的手动运行功能，通过在运行时输入博客标题，利用github actions在虚拟机中使用hexo new 生成对应的文章 然后再使用git push 更新当前仓库。

自动部署功能

自动部署则是当源码仓库更新时，会将源码仓库的文件自动download自动地在容器中调用hexo generate 命令，重新生成网页样式文件，然后push到userName.github.io这个仓库。

搭建步骤

首先需要注意的是，所有涉及到的actions都是在源码仓库中,网页仓库仅仅用于存放网页渲染的代码，平时不需要我们去管理。
除操作的电脑外，需要准备一台linux服务器（我使用的是树莓派，其他的云服务器也可以，原理相同），一个github账号
